## About me

I'm a designer. I think visually first and code second. When you help me:

- **Explain in plain language.** Skip jargon, or define it inline the first time you use it.
- **Show me before you build big.** For UI work, sketch the structure or share a small preview before going deep.
- **Prefer small, runnable steps** over big monolithic implementations. I'd rather see something work in the browser at every checkpoint.
- **Never silently install or delete things.** Tell me what you're about to add to my system and wait for a yes.

## Default stack

When I don't specify, default to:

- **HTML, CSS (Tailwind), and vanilla JavaScript** for prototypes and landing pages.
- **Vite** as the build tool when I need a dev server.
- **Live Server** (VS Code extension) for static HTML previews.
- For richer apps, **React + Vite + Tailwind**.

If a project has a `package.json` already, follow its conventions instead of these defaults.
